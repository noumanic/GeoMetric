Transformation Logic

The raw geojson has highly inconsistent variable naming and notorious "-99" ISO code bugs for nations like France and Norway. The pipeline manually patches these holes and strictly shrinks the 95 metadata columns down to only the 8 required.

# Fix known ISO code issues in Natural Earth (Critical manual patch)
fixes = {"France": "FRA", "Norway": "NOR", "Kosovo": "XKX", "Somaliland": "SOM"}
for name, iso in fixes.items():
    world.loc[world["country_name"] == name, "iso_a3"] = iso

# Drop lingering invalid geometries and standardize to pure unprojected spherical
world = world[world["iso_a3"] != "-99"]
world = world.set_crs("EPSG:4326", allow_override=True)
world.to_file(PATHS["processed"] / "world_countries.gpkg", driver="GPKG")

Transformation Logic

To avoid spatial mapping errors, all pseudo-countries (like "High-income countries" starting with OWID_) are dropped via string filtering. Critically, if `co2_per_capita` is `NaN` but total population exists, the pipeline dynamically calculates the ratio to prevent missing geographic blobs.

# Filter strictly to the analysis year (2020)
df = load_emissions(ANALYSIS_YEAR)

# Impute missing per-capita metrics (Fail-safe interpolation)
mask = df["co2_per_capita"].isna() & df["co2_total"].notna() & df["population"].notna()
df.loc[mask, "co2_per_capita"] = df.loc[mask, "co2_total"] * 1e6 / df.loc[mask, "population"]
df.to_csv("emissions_2020.csv", index=False)

4. Global Aviation Network

Source: OpenFlights Airports.dat & Routes.dat

Relational Fact Merge

RAW ISOLATED FILES

Airports: 7,000+ nodes (Has Coordinates)
Routes: 67,000+ edges (No Coordinates)
➔ PREPROCESS ➔

ENRICHED DIMENSIONS

Airports: Have 'total_routes' score

Transformation Logic

This executes classic dimensional data modeling. We group the raw Flight Lines (Fact Table) to count departures, merging that metric *into* the Airports (Dimension) to rank hub size. We then create a spatial dictionary to reverse-inject the Latitude/Longitude of the hubs *back into* the Flight Lines so matplotlib can plot Origin ➔ Destination lines physically.

# 1. Feature Engineering: Compute graph degree/centrality for each airport
src_count = routes.groupby("src_iata").size().reset_index(name="departures")
airports = airports.merge(src_count, left_on="iata", right_on="src_iata", how="left")
airports["total_routes"] = airports["departures"].fillna(0) + airports["arrivals"].fillna(0)

# 2. Spatial Enabing: Attach coordinates to raw routes for matplotlib line plotting
coord_map = airports.set_index("iata")[["lat", "lon", "country"]].to_dict("index")
routes["src_lat"] = routes["src_iata"].map(lambda x: coord_map.get(x, {}).get("lat", np.nan))
routes["dst_lon"] = routes["dst_iata"].map(lambda x: coord_map.get(x, {}).get("lon", np.nan))

6. Master Database Integration

The primary artifact used for parts 1 through 7 mapping analysis.

Master GeoPackage

Transformation Logic

The previous cleaned files are relationally joined via a `Left Join` on `iso_a3`. A crucial final safety net step establishes `pop_final`: if external World Bank population metrics missed a tiny nation, the spatial base-map estimate seamlessly steps in.

# Left join: Geometries + Emissions + Demographics
master = world.merge(emiss, on="iso_a3", how="left")
master = master.merge(pop, on="iso_a3", how="left")

# Final algorithmic imputation & feature generation
master["pop_final"] = master["population"].fillna(master["pop_est"])
master["area_km2"] = master.to_crs("+proj=cea").geometry.area / 1e6
master["pop_density"] = master["pop_final"] / master["area_km2"]

# Expose to analysts
master.to_file(PATHS["processed"] / "master_world.gpkg", driver="GPKG")
